# Viterbi decoder simulation

Решение для задачи 3: кодер и декодер Витерби для двоичного сверточного кода, модель двоичного симметричного канала и построение зависимости BER после декодирования от вероятности ошибки в канале.

## Сборка и запуск

```bash
make run
python3 plot.py
```

Или одной командой:

```bash
make plot
```

После запуска появляются:

- `results/ber.csv` - таблица `channel_error_probability,decoded_bit_error_rate,raw_channel_bit_error_rate,bits`;
- `results/ber.png` - график зависимости BER от вероятности ошибки в канале.

## Настройка кода

Порождающая матрица задается в `src/main.cpp`:

```cpp
ConvolutionalCode code(
    1,        // k: входных битов за такт
    2,        // n: выходных битов за такт
    6,        // memory: число задержек
    {{0o171}, {0o133}}
);
```

Для общего кода `k/n` нужно передать матрицу размера `n x k`, где каждый элемент - двоичная маска коэффициентов порождающего полинома. Бит `0` соответствует текущему входному биту, бит `d` - задержке на `d` тактов.

Пример для `k = 2`, `n = 3`, `memory = 2`:

```cpp
ConvolutionalCode code(
    2,
    3,
    2,
    {
        {0b111, 0b001},
        {0b101, 0b011},
        {0b001, 0b111}
    }
);
```

Декодер использует жесткие решения и метрику Хэмминга. В примере кодер добавляет хвост из `memory` нулевых входных символов, поэтому декодер выбирает нулевое конечное состояние.
